# WhatsApp Web Clone - Daily Development Report

## Day 1: Foundation & Backend Setup
- Initialized project structure (MERN stack).
- Configured Express server with Socket.IO integration.
- Connected to MongoDB with Mongoose models:
  - User: Username, email, password hashing, avatar, status tracking.
  - Message: Sender/receiver IDs, text content, real-time status (sent/delivered/seen).
  - Chat: Participants tracking and last message reference.
- Implemented core API routes:
  - Auth: Register and Login with JWT generation.
  - Users: Fetch contact list, update user profile.
  - Messages: Send/fetch message history.
  - Chats: Retrieve active chat lists for a user.
- Integrated Socket.IO for real-time connection and room joining.

## Day 2: Frontend Architecture & UI Development
- Initialized React project with Vite and Tailwind CSS.
- Configured Axios interceptor for automatic JWT header injection.
- Built core components:
  - Sidebar: Dynamic chat list and user search.
  - ChatWindow: Header, scrollable message area, and input.
  - MessageBubble: Conditional styling for sent vs received messages.
- Implemented global state management via Context API (ChatContext).
- Styled the app with WhatsApp's signature dark theme colors and custom scrollbars.
- Resolved dependency issues related to Tailwind CSS v4 and PostCSS.

## Day 3: Feature Enhancement & UX Polish
- Resolved CORS issues allowing cross-origin communication from dynamic dev ports.
- Implemented Advanced Sidebar Navigation:
  - Vertical tab bar for Chats, Status, Channels, Communities, and Settings.
- Added Floating New Chat Button:
  - Implemented the green "+" button and a dedicated New Chat drawer for discovering contacts.
- Developed Profile Management:
  - Interactive profile drawer allowing users to update display names and avatars.
- Enhanced Real-time Messaging:
  - Implemented "Message Seen" (blue double checks) status via Socket.IO.
  - Added real-time "Typing..." indicators.
- Implemented Group Chat:
  - Backend support for group models and controllers.
  - Frontend "New Group" creation modal with multi-user selection.
- Developed Status Feature:
  - Backend "Status" model and routes.
  - Frontend Status upload drawer and viewing interface.
- Added Contextual Menus:
  - Functional 3-dots menu in both Sidebar and Chat Window with options like "Clear Chat", "New Group", and "Contact Info".
- Accessibility & Best Practices:
  - Fixed form field accessibility by adding unique IDs, name attributes, and associated labels across all forms (Login, Profile, New Group, Status Upload, and Message Input).
- Final code audit and push to GitHub repository.

## Day 4: Bug Fixing, Performance, and Finalization
- **File/Image Upload**: Implemented end-to-end file and image uploading with Multer, including frontend previews and loading states.
- **Emoji Picker**: Fixed the emoji picker to close on outside click and insert emojis at the correct cursor position.
- **Search Functionality**: Implemented debounced search for both sidebar user filtering and in-chat message searching with highlighted results.
- **Pagination**: Added infinite scroll to the chat window to paginate through message history.
- **Error Handling**: Added robust error handling on both the frontend and backend to prevent crashes and provide user feedback.
- **Defensive Coding**: Refactored the codebase to include defensive coding practices, such as optional chaining and default fallback values.
- **Performance Improvements**: Optimized component rendering with `React.memo` and debounced inputs.
- **Socket.IO Cleanup**: Ensured proper cleanup of Socket.IO listeners to prevent memory leaks.
- **UI Improvements**: Added loading indicators for sending messages and a "No messages yet" empty state.
- **Documentation**: Created a `TESTING.md` file with detailed testing instructions.
- **Git Workflow**: Staged and committed all changes with a meaningful commit message.
